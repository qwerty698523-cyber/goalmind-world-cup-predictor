# GoalMind 世界杯预测网站

这是一个 2026 世界杯预测网站原型，包含实时赛程同步、球队动态战力、赛前预测、赛后复盘、淘汰赛模拟、天气与伤病变量、模型评价指标和发布前检查。

当前版本适合先作为“公开测试版”发布：页面、接口和数据链路已经可以运行；模型评价系统已经接入；但模型质量还没有达到专业合格线，尤其是平局识别、概率校准、前三候选比分覆盖率和样本外测试规模仍需继续改进。

## 本地打开

双击：

```text
启动网站.cmd
```

或在 Windows 命令提示符中运行：

```cmd
cd /d C:\Users\gale\Documents\世界杯
start-site.cmd
```

默认地址是：

```text
http://127.0.0.1:3000
```

如果 3000 端口被占用，启动脚本会自动尝试后续端口。黑色服务窗口需要保持开启，关闭后本地网站也会停止。

## 发布前检查

发布前建议先运行：

```cmd
cd /d C:\Users\gale\Documents\世界杯
npm run release:check
```

这个命令会做四件事：

1. 检查核心代码语法和评价函数测试；
2. 同步最新赛程数据并刷新 `live-cache.json`；
3. 生成离线兜底数据 `fallback-data.js`；
4. 输出 `release-report.json`，记录当前是否适合作为测试版发布，以及模型哪些指标未达标。

注意：`model:gate` 当前可能失败。这不代表网站打不开，而是代表模型还没有达到你设定的正式合格线。公开测试版可以发布，但页面和说明中不应宣称“专业级准确”。

## 发布到互联网

双击：

```text
登录并发布网站.cmd
```

或运行：

```cmd
cd /d C:\Users\gale\Documents\世界杯
publish-site.cmd
```

脚本会使用 Vercel 发布正式网站。首次发布时需要登录 Vercel/GitHub。发布完成后，公网地址会写入：

```text
deployment-url.txt
```

这个公网地址可以被其他电脑和手机访问。搜索引擎收录通常不是即时的，发布后还需要把域名提交到 Google Search Console、Bing Webmaster Tools 或百度搜索资源平台。

## 已实现功能

- 2026 世界杯赛程、比分、状态和球队数据同步；
- 48 支球队动态战力；
- 基于 Elo、近期状态、进攻、防守、比赛质量、天气、伤病和阵容可用性的赛前预测；
- 90 分钟常规时间胜、平、负概率；
- 最可能比分、前三候选比分、双方预期进球、总进球和双方进球概率；
- 淘汰赛区分“90 分钟赛果概率”和“最终晋级概率”；
- 若淘汰赛 90 分钟打平，页面会说明加时赛和点球大战路径；
- 完赛后显示最终比分旁的赛前预测；
- 预测结果按颜色标记：比分和胜负都正确为绿色，只猜对胜负为黄色，方向错误为红色；
- 单场比赛页包含赛前预测、赛后总结、预测复盘和模型学习信息；
- 球队档案包含名单、教练组、首发/替补、阵型、人员结构、有球阶段和无球阶段分析；
- 小组出线模拟和淘汰赛路径模拟；
- 模型评价指标：命中率、Log Loss、Brier Score、ECE、平局召回率、宏平均 F1、精确比分命中率、前三比分覆盖率、进球误差等；
- 基准模型对比：随机基准、固定比分基准、基础 Elo、Elo+Poisson。

## 常用命令

```cmd
npm run check
npm run model:test
npm run model:replay
npm run model:gate
npm run release:check
```

其中：

- `check`：检查 JavaScript 语法；
- `model:test`：运行模型评价相关单元测试；
- `model:replay`：用当前模型重放已完赛比赛；
- `model:gate`：按正式模型质量门槛检查，当前不保证通过；
- `release:check`：发布前检查，允许作为测试版发布但会记录模型未达标项。

## 数据来源

- 实时赛程和比赛详情：ESPN 公开接口；
- 历史国家队比赛：International football results 数据集；
- 战力和预测：GoalMind 本地模型；
- 天气：按比赛场馆位置读取小时级天气；
- 阵容和伤病：优先使用公开名单与 ESPN roster 信息，缺失时回退本地估计。

预测仅供参考，不应使用“稳赢”“必胜”“百分百准确”等表述。
